Project Snapshot

Local Gemma Agent

이 프로젝트는 Python + Tkinter + Ollama + Gemma 3로 만든 로컬 AI 에이전트입니다. 외부 SaaS 없이 PC 내부에서 추론하고, 웹 검색·로컬 문서 검색·Obsidian vault 검색을 하나의 에이전트 루프에 묶은 구조입니다.

Python 3.13 Tkinter GUI Ollama API Gemma 3 4B PowerShell Scripts Local-First

Runtime Layers

UI Layer

Tkinter, ttk, scrolledtext, threading 기반 데스크톱 채팅 인터페이스

Agent Layer

프롬프트 기반 tool-routing, JSON 파싱, 대화 상태 유지

Tool Layer

파일 읽기/쓰기, 웹 검색, 로컬 문서 검색, Obsidian 노트 검색

Model Layer

http://127.0.0.1:11434의 Ollama와 gemma3:4b

UI Notes

  • 좌측은 대화에 집중한 메인 영역
  • 우측은 현재 세션과 퀵 액션 패널
  • 응답 생성은 백그라운드 스레드에서 수행
  • 입력창은 멀티라인, Ctrl+Enter 전송
  • 툴 호출은 내부적으로만 수행되고 사용자는 자연어로만 상호작용

기술 스택 요약

언어

Python 3.13

패키징

pyproject.toml, editable install, 표준 라이브러리 중심

모델 런타임

Ollama 로컬 서버 + Gemma 3 4B

GUI

Tkinter, ttk, scrolledtext

실행 스크립트

PowerShell bootstrap.ps1, run-agent.ps1, run-gui.ps1

검색/RAG

간단한 토큰 점수 기반 로컬 문서 검색 + DuckDuckGo HTML 검색 파싱

지식 소스

workspace/, workspace/docs, Obsidian vault

보안 방향

기본 추론은 로컬, 파일 쓰기는 workspace 범위로 제한

Data Flow Wireframe

1. User Input

GUI 입력창 또는 CLI 프롬프트로 자연어 요청 입력

2. Agent Prompt

시스템 프롬프트에 workspace/docs/vault 정보와 도구 카탈로그 주입

3. Ollama Chat

Gemma 3가 툴 호출 JSON 또는 최종 응답 메시지 생성

4. Tool Execution

파일, 웹 검색, RAG, Obsidian 검색을 Python 툴로 실행

5. Final Answer

도구 결과를 다시 모델에 전달하고 사용자에게 응답 표시

현재 장점

가볍고 설치가 쉽고, 외부 API 키 없이 로컬에서 바로 테스트 가능

현재 한계

툴 호출은 프롬프트 기반이며, RAG는 임베딩 DB 없이 단순 점수 방식

다음 확장

대화 저장, 모델 선택기, 툴 로그 패널, PDF 파싱, 벡터 검색 추가 가능
